🔔 نظام الإشعارات داخل المنصة (Notification System)

Business Specification Document (BRD)

🔷 1. مقدمة

نظام الإشعارات هو أحد الأنظمة الأساسية داخل المنصة، ويهدف إلى:

  • تحسين تجربة المستخدم (عميل – بائع – إدارة المنصة)
  • إرسال أحداث مهمة في الوقت المناسب
  • ضمان متابعة الطلبات والعروض والمعاملات
  • تقليل الأخطاء التشغيلية
  • رفع معدلات التفاعل والمبيعات

النظام يدعم الإشعارات عبر عدة قنوات: Email – SMS – Push – In-App

🔷 2. فئات الإشعارات (Notification Categories)

تنقسم الإشعارات إلى أربع فئات رئيسية:

🟦 2.1 إشعارات العملاء (Customer Notifications)

  • ✅ تأكيد الطلب
  • 📦 تحديث حالة الشحن
  • 🚚 إشعار وصول مندوب التوصيل
  • 💳 نجاح الدفع أو فشله
  • 🎫 إنشاء تذكرة دعم
  • 💰 رصيد المحفظة
  • 💸 استرجاع الأموال (Refund)
  • 🔄 اكتمال عملية الإرجاع
  • 🎁 اشعارات الخصومات والعروض

🟩 2.2 إشعارات البائعين (Seller Notifications)

  • 🛍️ وصول طلب جديد
  • 📉 نقص المخزون
  • ⚠️ تعليق منتج / تعليق عرض
  • ✅ الموافقة على طلب المشاركة في حملة
  • ❌ رفض طلب المشاركة
  • 💰 تحويل أرباح البائع إلى المحفظة
  • 🔄 طلب إرجاع من العميل
  • ⚖️ تذكرة نزاع جديدة
  • 📝 قبول/رفض تغيير بيانات متجره

🟧 2.3 إشعارات الإدارة (Admin Notifications)

  • 👤 بائع جديد في انتظار الموافقة (KYC)
  • 🎯 طلب حملة جديد من بائع
  • 💵 طلب سحب رصيد
  • ⚖️ شكوى (Dispute) من عميل
  • ✏️ طلب تعديل منتج
  • 🚨 Report Abuse على منتج

🟫 2.4 إشعارات النظام (System Alerts)

  • ❗ Errors
  • 💳 Payment Gateway Failure
  • ⚡ Server Overload
  • 📊 Stock Sync Failure
  • 🔒 API Rate Limit Reached

🔷 3. قنوات الإشعارات (Notification Channels)

النظام يدعم إرسال الإشعار عبر عدة مسارات:

القناة الوصف الاستخدام
📱 In-App داخل التطبيق تظهر داخل المنصة (الويب/الموبايل)
📧 Email البريد الإلكتروني إنشاء حساب، تأكيد الطلب، استرجاع كلمة المرور
📲 SMS الرسائل النصية OTP، الدفع عند الاستلام، حالة التوصيل
🔔 Push (Mobile) إشعارات الموبايل للعملاء والبائعين
🌐 Web Push إشعارات المتصفح للمتصفحات الداعمة
📌 ملحوظة:

يمكن إرسال الإشعار على قناة واحدة أو عدة قنوات حسب الأهمية.

🔷 4. هيكل الإشعار (Notification Structure)

كل إشعار يجب أن يحتوي على:

الحقل الوصف
NotificationId رقم الإشعار
UserId صاحب الإشعار
Type النوع (Order / Seller / System)
Severity درجة الأهمية
DeliveryChannel قناة الإرسال
Title عنوان الإشعار
Message نص الإشعار
Data بيانات إضافية منسقة JSON
IsRead مقروء/غير مقروء
CreatedAt وقت الإرسال
SentAt وقت التوصيل
Status Delivered / Failed / Pending

🔷 5. مستويات الإشعار (Notification Priority)

🟥 High Priority

  • فشل دفع
  • OTP
  • طلب جديد للبائع
  • إلغاء طلب
  • استرجاع أموال

🟧 Medium Priority

  • تحديث حالة الطلب
  • تحديث حالة الشحن
  • الموافقة على طلبات

🟩 Low Priority

  • عروض وتسويق
  • تنبيهات عامة
  • نصائح وإعلانات
